Git指令

  • Notes

命令关键词，理解工作区、暂存区、版本库。

$ git config –global user.name “my name”
$ git config –global user.email “my_email@myemail.com“ // 设置

远程仓库类：git clone / remote / pull / push / rebase / fetch

$ git clone username@host:/path/to/repository // 克隆仓库
$ git remote add origin git@github.com:ouyoungs7/myRepo.git // 添加远程地址：红字是刚新建的仓库
$ git pull // 更新本地仓库至最新改动（更新=获取fetch+合并merge 远端的改动）
$ git push origin master // ③提交远程仓库（推送改动）
$ git fetch origin // 到服务器获取最新的版本历史
 

本地常规操作：git add / commit / rm / merge / reset

$ git mkdir // 新建仓库
$ git init // 初始化仓库
$ git add * // 添加到暂存区
$ git add // 手动标记其合并成功
$ git commit -m “代码提交信息” // 提交到HEAD
$ git merge // 合并分支（若冲突失败，需要手动合并）
$ git reset –hard origin/master // 本地主分支

 

分支管理类： git branch / tag / checkout /stash

$ git branch -d feature_x // 删除分支feature_x
$ git tag 1.0.0 1b2e1d63ff
$ git checkout -b feature_x // 创建分支feature_x，并切换过去
$ git checkout master // 切换回主分支

 

信息查询类：git status / log / diff

$ git status // 展示工作区和stage区状态
$ git log
$ git diff <source_branch> <target_branch> // 预览差异

 

其他类：

$ gitk // 内建的图形化
$ git config color.ui true // 彩色git输出
$ git config format.pretty online // 单行显示历史记录
$ git add -i // 交互式添加文件到暂存区
 

Github简明教程：http://www.runoob.com/w3cnote/git-guide.html
