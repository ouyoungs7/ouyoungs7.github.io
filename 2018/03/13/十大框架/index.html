十大框架

  • Notes

技术为解决问题而生
要成为前端工程师，而不是表单工程师！

随着人们对前端性能的要求、项目复杂度增大，要求对前端资源打包、压缩、合并、模块化。
// grunt、gulp、webpack

Angular2、Vuejs、React、BackBone

1 十大框架介绍

源网址：https://www.cnblogs.com/kingboy2008/p/5261771.html
需要相关资料，进入该博客，找对应链接地址即可。
 
简单记录如下：

1.1 Bootstrap

界面和谐，易上手。

1.2 Amaze UI

“基于社区开源项目构建的一个跨屏前端框架，以移动优先，从小屏到大屏，最终实现所有屏幕适配，适应移动互联潮流”
加入了更多符合中国市场特性的元素，针对跨屏适配和兼容性。

1.3 Frozen UI

QQ会员前端，社区活跃，组件丰富。

1.4 WeUIi

在github上，框架及其简单，模块7个左右。体量小口碑好。适合微信公众。

1.5 SUI

“一套基于Bootstrap开发的前端组件库，同时也是一套设计规范。”用过Bootstarp，可轻松转向SUI。

1.6 AUI

1.7 MUI

以IOS为主题设计，也补充了android特有的UI样式。

1.8 Semantic UI

1.9 Foundation

1.10 UiKit

1.11 Pure

2 前端框架区分

源网页自己翻：https://www.zhihu.com/search?type=content&q=%E5%89%8D%E7%AB%AF%E6%A1%86%E6%9E%B6

2.1 纯页面皮肤

(HTML+CSS)：Dreamwearver

2.2 UI皮肤+少量js

Bootstrap加入了html5/css3特性，对响应式支持及良好的体验。封装度较低

2.3 大杂烩工具包+案例库

metronic/adminLTE系列集成了各种jQuery插件、富文本编辑器等js库。

2.4 重度封装

jQuery-ui通过框架暴露的接口进行开发，更适合传统管理软件的开发。

2.5 重度js框架

“富客户端框架”，easy-ui/DWZ

2.6 庞然大物

extjs自成体系的元素选择引擎和浏览器兼容方案，几乎不必学习任何前端知识就可以做出”很漂亮的”前端页面
离开extjs几乎不会写前端页面。

2.7 封装度最高

用后台语言写前端，但前后台耦合更紧密，团队开发灾难。
GWT有google背书。

“计算机这门学科就是这样，当一件事物复杂到一定程度就一定有人抽象出各种框架来提高可维护性与拓展性”。
传输复杂：osi 7层和5层框架
服务器集群复杂：云计算框架
